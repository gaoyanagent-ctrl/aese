# 快速 2D 企业沙盘实施计划

## 1. 目标与工期

在 **3 到 4 个工作日**内交付可访问的 2D 企业沙盘预览版。它以华辰热管理系统集团、苏州制造基地、电池冷却板 A 线和订单加急七幕故事为唯一范围，用 22 个确定性事件驱动画布、KPI、事件流和 Agent 建议。

本计划是当前唯一 active plan。M4 异常事件在线化和 M5 Agent Runtime 在此版本完成后恢复推进。

## 2. 完成定义

- 本地开发服务可访问，进入即显示实际沙盘工作台。
- 22 个事件可播放、暂停、前后单步、倍速和重置。
- 2D 画布完整展示 A 线工序、仓库、供应和发运路径。
- 订单、库存、设备、质量、生产和发运状态随事件确定性变化。
- 事件面板、对象详情、五项 KPI 和三类 Agent 建议可用。
- 静态数据通过 `ScenarioDataSource` 提供，不硬编码在 React 组件中。
- 单元/组件测试通过；Playwright 在桌面和移动视口通过，画布非空且无明显重叠。
- README、code map、运行说明和 progress log 同步。

## 5. 测试清单

功能测试：

- 初始状态显示 22 个待播放事件和正确场景时间。
- 播放/暂停不会跳过或重复事件。
- 上一步、下一步、重置和倍速行为稳定。
- 设备停机时对应设备与工作中心进入严重状态。
- 来料不良时检验批隔离，质量 Agent 建议出现。
- 最终发运数量、短缺数量和交付风险与场景合同一致。

视觉测试：

- 画布节点和边在首次加载及尺寸变化后均可见。
- 控制栏、KPI、事件面板和抽屉不互相覆盖。
- 最长中文对象名、事件名和建议文本不溢出容器。
- 移动端可以在画布、事件、Agent 和详情之间切换。

边界测试：

- `preview.json` 缺失、格式错误或事件为空时显示明确错误。
- 第 0 步不能继续后退，第 22 步自动停止。
- 切换速度、切换标签页和打开详情不会重置播放状态。

## 6. 后续衔接

预览版完成后按以下顺序接入真实能力：

1. M4 实现 IAOS simulation ingress，接入供应商延期、设备停机和来料不良。
2. 实现 `IaosScenarioDataSource`，使用 IAOS 快照 API 与 SSE/WebSocket 替代静态事件源。
3. M5 接入真实计划、质量和经营分析 Agent 输出及治理状态。
4. 将验证成熟的沙盘组件集成到 IAOS 正式前端；AESE 继续保留场景预览和回归用途。

## 7. 风险控制

- **看起来能动但不真实**：所有画面变化必须绑定现有对象编码、事件 ID 和 expected outcomes。
- **预览逻辑变成第二套业务引擎**：浏览器只应用已定义 delta，不计算 MRP、排产、质量判定或审批。
- **后续接真数据重写**：组件只依赖 `SandboxScenario`，静态 JSON 和 IAOS DTO 均通过 adapter 转换。
- **画布开发拖延**：首版使用规则网格与稳定节点，不做自由拖拽编辑器、3D、粒子效果和复杂动画。
