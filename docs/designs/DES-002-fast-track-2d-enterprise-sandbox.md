# 快速 2D 企业沙盘设计

## 1. 目标

用现有华辰场景包快速交付一个可播放、可检查的 2D 企业运行视图。用户进入页面后直接看到苏州制造基地、电池冷却板 A 线、当前经营状态和事件推进，不设置营销落地页。

首版回答四个问题：

1. 企业当前发生了什么？
2. 订单、物料和生产任务处于哪里？
3. 哪些异常正在影响交付？
4. Agent 给出了什么建议，依据是什么？

## 2. 用户和核心任务

主要用户是产品设计者、实施顾问、销售演示人员和 IAOS 研发人员。核心任务是播放第一条故事、定位风险对象、查看事件因果链、核对 KPI 变化和复述 Agent 建议。

## 3. 首屏布局

桌面端采用稳定的工作台布局：

```text
┌──────────────── 场景控制栏：时间 / 播放 / 单步 / 倍速 / 重置 ────────────────┐
├────────────┬──────────────────────────────────┬────────────────────────┤
│ 企业导航   │ 2D 苏州基地 / 电池冷却板 A 线    │ 事件流 / Agent 建议     │
│ 集团/基地  │ 工作中心、设备、订单与物流高亮   │ 当前事件、因果与建议    │
├────────────┴──────────────────────────────────┴────────────────────────┤
│ KPI：订单需求 / 可用成品 / 缺料风险 / 产能风险 / 交付风险                 │
└─────────────────────────────────────────────────────────────────────────┘
```

- 顶部控制栏：场景名称、Preview 状态、当前仿真时间、播放/暂停、上一步、下一步、速度、重置。
- 左侧导航：集团、苏州制造基地、电池冷却板 A 线及关键仓库；点击后聚焦对应对象。
- 中央画布：原料仓、成形、机加工、焊接、清洗、检漏、总装、包装、成品仓和发运区。
- 右侧面板：当前事件、最近事件、严重级别、关联对象、计划/质量/经营 Agent 建议。
- 底部 KPI：固定高度，数值变化不引发布局跳动。
- 对象详情：点击节点后从侧边抽屉展示编码、状态、数量、风险和相关事件。

移动端保留场景控制和画布，企业导航、事件流、详情通过 Tabs/Drawer 切换；不要求在窄屏同时显示三栏。

## 4. 视觉语义

- 正常运行：中性灰与绿色状态。
- 关注：琥珀色，表示交期、库存或质量风险。
- 严重：红色，表示停机、拒收或确定性短缺。
- 信息：蓝色，用于订单、物料流和选中对象。
- 节点状态用图标、文字和颜色共同表达，不能只依赖颜色。
- 中央画布优先表达真实工艺和物流路径，不使用装饰性图形替代业务对象。

## 5. 七幕播放合同

| 幕 | 关键画面变化 | 关键输出 |
| --- | --- | --- |
| 1 客户追加订单 | 客户到销售订单路径高亮，需求增加 | 交付风险从正常转为关注 |
| 2 初次 MRP | BOM 物料和库存节点联动 | 缺料清单、采购与生产建议 |
| 3 供应商延期 | 供应商到原料仓路径变为风险状态 | 到料延误及交期影响 |
| 4 设备停机 | 对应工作中心/设备变红，工序流受阻 | 产能缺口和重排建议 |
| 5 备选供应与来料不良 | 备选供应路径出现，检验区报警 | 隔离批次和补料建议 |
| 6 生产与入库 | 工单沿 A 线推进，成品仓数量增加 | 产出、良率和剩余缺口 |
| 7 分批发运与复盘 | 成品仓到客户路径推进 | 实发 2,700、短缺 300、管理摘要 |

22 个种子事件按时间顺序驱动状态变化。首版使用预先计算的 delta/snapshot，不在浏览器中实现 MRP、排产或离散事件引擎。

## 6. 数据合同

新增 `scenario-packs/hctm/stories/order-expedite-01/preview.json`，只保存视图所需数据：

- `scenario`：名称、版本、起止时间、默认速度。
- `layout`：区域、节点、连线及稳定业务编码。
- `entities`：可点击对象的摘要和初始状态。
- `timeline`：22 个事件及其 state delta、visual cue、关联对象。
- `kpis`：每一步的订单、库存、缺料、产能和交付风险快照。
- `agent_outputs`：Agent、建议、依据、置信度和是否需要人工确认。

前端只依赖内部 `SandboxScenario` 视图模型。数据源接口：

```ts
interface ScenarioDataSource {
  loadScenario(key: string): Promise<SandboxScenario>;
}
```

首版实现 `StaticScenarioDataSource`；后续实现 `IaosScenarioDataSource`，通过 IAOS API 获取快照并通过 SSE/WebSocket 接收事件。

## 7. 技术方案

- React + TypeScript + Vite。
- `@xyflow/react` 构建 2D 工厂与业务流画布。
- Lucide React 提供操作图标。
- Vitest + Testing Library 验证状态推进和组件交互。
- Playwright 验证桌面与移动端完整故事和视觉布局。
- CSS 变量维护语义色、间距和稳定尺寸，避免引入重型设计系统。

播放内核采用纯 reducer：`initial state + ordered event deltas -> current state`。跳转到任意步骤时从初始状态确定性重放，保证重置、前进、后退和测试结果一致。

## 8. 非目标

- 不实现通用离散事件仿真引擎。
- 不调用真实 LLM 或自动执行 Agent 建议。
- 不提供业务 CRUD、审批、权限管理和配置后台。
- 不实现 3D 厂区、数字人或复杂物理动画。
- 不在首版接入全部 IAOS 在线对象和 18 类正式事件。

## 9. 验收标准

- 页面打开即显示苏州基地 A 线，不出现空白画布或落地页。
- 可播放、暂停、前后单步、倍速和重置，并可抵达全部 22 个事件。
- 七幕中的订单、供应、设备、质量、生产和发运变化在画布可见。
- 点击关键节点可查看对象详情和关联事件。
- KPI 与故事最终结果一致，最终明确显示实发 2,700、短缺 300。
- 三类 Agent 输出可在相关事件处查看，且区分建议与已执行动作。
- 桌面和移动视口无文本溢出、控件重叠或布局跳动。
- 前端无需后端即可运行，数据源边界允许后续接 IAOS API/SSE。
