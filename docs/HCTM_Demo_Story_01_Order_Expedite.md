# HCTM Demo Story 01 - Order Expedite

本文定义 AESE 第一条可执行演示故事：**客户追加订单下的交付承诺重算**。它面向产品、研发、销售演示和后续 agent，用于说明如何把华辰热管理系统集团的主数据、事件流、Agent 输出和 IAOS 页面串成一次完整演示。

## 1. 故事定位

故事名称：

- 客户追加订单下的交付承诺重算。

故事编码：

- `hctm_order_expedite_01`

租户：

- `tenant-hctm`

主对象：

- 销售订单：`SO-202607-0001`
- 追加订单：`SO-202607-0001-ADD1`
- 产品：`HCTM-BCP-A01`
- 工厂：`PLT-SZ`
- 产线：`SZ-BCP-LINE-A`

演示目标：

- 展示 IAOS 能从客户订单触发订单到交付事件链。
- 展示供应商延期和设备故障如何改变计划风险。
- 展示计划 Agent、质量 Agent、经营分析 Agent 如何基于业务上下文输出建议。
- 展示管理层能看到交付、成本、库存、质量和产能影响。

## 5. 分幕演示流程

### 5.1 第一幕：客户追加订单

业务背景：

- 星河新能源汽车原订单 10,000 件。
- 客户临时追加 2,000 件。
- 总需求变为 12,000 件，交期仍为 `2026-07-20`。

用户操作：

1. 销售经理打开订单视图。
2. 创建或展示 `SO-202607-0001` 原订单。
3. 创建或展示 `SO-202607-0001-ADD1` 追加订单。
4. 确认总需求 12,000 件。

系统事件：

| seq | event_id | type | 说明 |
| --- | --- | --- | --- |
| 1 | `evt-20260701-0001` | `o2d.order.received` | 收到 10,000 件原订单 |
| 2 | `evt-20260701-0002` | `o2d.order.received` | 收到 2,000 件追加订单 |
| 3 | `evt-20260701-0003` | `o2d.order.confirmed` | 确认总需求 12,000 件 |

页面应展示：

- 客户：星河新能源汽车有限公司。
- 产品：电池冷却板组件 A01。
- 总需求：12,000 件。
- 交期：`2026-07-20`。
- 订单状态：`confirmed`。

讲解要点：

- 这不是静态 demo 数据，订单确认会进入事件流。
- `o2d.order.confirmed` 是 IAOS 当前 O2D 场景可以消费的关键事件。

### 5.2 第二幕：初始 MRP 和供应计划

业务背景：

- 成品库存已有 1,200 件。
- 净生产需求为 10,800 件。
- 铝板总需求约 12,600 张。
- 当前库存 8,000 张，主供应商在途 5,000 张。

用户操作：

1. 计划员打开 MRP / 计划视图。
2. 查看订单确认后生成的初始 MRP。
3. 查看采购需求和生产订单建议。

系统事件：

| seq | event_id | type | 说明 |
| --- | --- | --- | --- |
| 4 | `evt-20260701-0004` | `o2d.mrp.generated` | 初始 MRP 生成 |
| 5 | `evt-20260701-0005` | `o2d.purchase_requirement.created` | 创建铝板时点风险采购需求 |
| 6 | `evt-20260701-0006` | `o2d.purchase_order.released` | 主供应商 PO 发布 |

页面应展示：

| 指标 | 值 |
| --- | --- |
| 总需求 | 12,000 件 |
| 成品库存 | 1,200 件 |
| 净生产需求 | 10,800 件 |
| 铝板需求 | 12,600 张 |
| 当前铝板库存 | 8,000 张 |
| 在途铝板 | 5,000 张 |
| 初始交付风险 | 低到中 |

计划 Agent 输出：

```text
订单总需求 12,000 件，扣除成品库存 1,200 件后需生产 10,800 件。
铝板数量上可覆盖需求，但依赖主供应商 5,000 张在途批次按期到达。
建议释放主供应商采购订单，并准备生产订单 WO-202607-0001。
```

讲解要点：

- 初始计划看起来可行，但对供应商 ETA 和焊接产能有依赖。
- 系统已经把后续风险暴露为可监听的上下文。

### 5.3 第三幕：供应商延期

业务背景：

- 主供应商 `SUP-ALPHA-AL` 原 ETA 为 `2026-07-08`。
- 供应商通知延期到 `2026-07-11`。
- 延期 3 天影响铝板可用时间。

用户操作：

1. 采购员打开采购视图。
2. 更新 `PO-202607-0001` 的最新 ETA。
3. 或由仿真器注入供应商延期事件。

系统事件：

| seq | event_id | type | 说明 |
| --- | --- | --- | --- |
| 7 | `evt-20260702-0001` | `o2d.supplier_shipment.dispatched` | 供应商发货 |
| 8 | `evt-20260708-0001` | `o2d.supplier_delivery.delayed` | 供应商延期 3 天 |

页面应展示：

- `PO-202607-0001` 状态：`delayed`。
- 原 ETA：`2026-07-08`。
- 新 ETA：`2026-07-11`。
- 影响订单：`SO-202607-0001`。
- 交付风险上升。

计划 Agent 输出：

```text
主供应商铝板 PO-202607-0001 延期 3 天。数量缺口不明显，但物料可用时点存在风险。
建议评估备选供应商 SUP-BETA-AL 的 2,000 张加急供货，同时保留主供应商到货计划。
```

经营分析 Agent 输出：

```text
供应商延期会提高准时交付风险。若不调整，后续生产窗口被压缩，可能需要加班或分批发运。
```

讲解要点：

- 系统不是只记录延期，而是将延期纳入同一订单链路。
- Agent 能基于 `correlation_id` 汇总订单、采购和库存上下文。

### 5.4 第四幕：关键设备故障

业务背景：

- 焊接工作中心是瓶颈。
- `LAS-WLD-02` 停机 8 小时。
- 当日焊接能力下降约 35%。

用户操作：

1. 设备工程师打开产线或设备视图。
2. 记录 `LAS-WLD-02` 设备故障。
3. 或由仿真器注入 `eam.machine.down`。

系统事件：

| seq | event_id | type | 说明 |
| --- | --- | --- | --- |
| 9 | `evt-20260708-0002` | `eam.machine.down` | 焊接设备停机 8 小时 |
| 10 | `evt-20260708-0003` | `o2d.mrp.generated` | 异常触发 MRP 重算 |

页面应展示：

- 设备 `LAS-WLD-02` 状态：`down`。
- 预计维修完成：`2026-07-08T17:30:00+08:00`。
- 瓶颈工作中心：`WLD-02`。
- 产能风险：高。
- MRP 重算原因：`exception_replan`。

计划 Agent 输出：

```text
供应商延期和 LAS-WLD-02 停机同时发生，当前交付风险升高。
建议组合方案：释放备选供应商 PO-202607-0002，启用加班 SHIFT-OT，并采用 9,000 件 + 3,000 件分批发运。
```

经营分析 Agent 输出：

```text
推荐方案会增加备选供应商采购成本、加班成本和检验成本，但能将交付风险从高降为可控。
```

讲解要点：

- 单一异常可能可控，多个异常叠加后需要重新决策。
- AESE 用事件链模拟企业真实运行，而不是预置一张静态结果表。

### 5.5 第五幕：备选供应商和质量风险

业务背景：

- 计划建议释放备选供应商 `SUP-BETA-AL` 的 2,000 张铝板 PO。
- 备选供应商首批来料需要加严检验。
- 检验发现 300 张表面划伤。

用户操作：

1. 采购员释放 `PO-202607-0002`。
2. 仓库收货 `GR-202607-0002`。
3. 来料检验员执行 `IQC-202607-0002`。
4. 质量工程师查看质量 Agent 建议。

系统事件：

| seq | event_id | type | 说明 |
| --- | --- | --- | --- |
| 11 | `evt-20260708-0004` | `o2d.purchase_order.released` | 备选供应商 PO 释放 |
| 15 | `evt-20260712-0001` | `whs.material.received` | 备选供应商到货 |
| 16 | `evt-20260712-0002` | `qms.incoming_inspection.failed` | 发现 300 张表面划伤 |

页面应展示：

- `PO-202607-0002` 状态：`released` 或 `received`。
- 检验单：`IQC-202607-0002`。
- 缺陷：`SURFACE_SCRATCH`。
- 不合格数量：300 张。
- 建议动作：隔离、禁止不合格数量投产、评估合格数量。

质量 Agent 输出：

```text
备选供应商 SUP-BETA-AL 首批铝板触发加严检验。
IQC-202607-0002 发现 300 张表面划伤，建议隔离 BETA-20260712-01 批次中的不合格数量，禁止直接投产。
剩余合格数量可在质量放行后用于紧急生产，但需要保留批次追溯。
```

讲解要点：

- 计划方案引入了质量风险，质量 Agent 负责约束和解释。
- Agent 之间不是各自说话，而是围绕同一业务链路协作。

### 5.6 第六幕：生产执行和成品入库

业务背景：

- 生产订单 `WO-202607-0001` 释放。
- 焊接工序开始并完成。
- 成品入库 10,500 件。

用户操作：

1. 计划员释放生产订单。
2. 班组长开始焊接工序任务。
3. 班组长完成工序报工。
4. 仓库完成成品入库。

系统事件：

| seq | event_id | type | 说明 |
| --- | --- | --- | --- |
| 17 | `evt-20260713-0001` | `proc.production_order.released` | 生产订单释放 |
| 18 | `evt-20260714-0001` | `proc.operation.started` | 焊接工序开始 |
| 19 | `evt-20260717-0001` | `proc.operation.completed` | 焊接工序完成，报废 80 件 |
| 20 | `evt-20260718-0001` | `whs.finished_goods.received` | 成品入库 10,500 件 |

页面应展示：

- 生产订单状态：`completed` 或接近完成。
- 焊接任务完成数量：10,520。
- 报废数量：80。
- 成品入库：10,500 件。
- 可发货库存：初始 1,200 + 新入库 10,500 = 11,700 件。

经营分析 Agent 输出：

```text
当前可发货库存达到 11,700 件，距离总需求 12,000 件仍有 300 件差距。
建议先发 9,000 件，剩余 3,000 件结合后续完工或客户允许窗口完成第二批发运。
```

讲解要点：

- 成品入库不只是库存数字变化，也影响交付承诺。
- 经营分析 Agent 从生产、库存和订单合并解释。

### 5.7 第七幕：分批发运和管理层复盘

业务背景：

- 方案采用分批发运。
- 第一批 9,000 件在 `2026-07-19` 发运。
- 第二批 3,000 件在 `2026-07-20` 发运。

用户操作：

1. 仓库主管创建或确认第一批发货。
2. 仓库主管创建或确认第二批发货。
3. 工厂厂长打开经营驾驶舱。

系统事件：

| seq | event_id | type | 说明 |
| --- | --- | --- | --- |
| 21 | `evt-20260719-0001` | `o2d.shipment.dispatched` | 第一批发货 9,000 件 |
| 22 | `evt-20260720-0001` | `o2d.shipment.dispatched` | 第二批发货 3,000 件 |

页面应展示：

| 指标 | 结果 |
| --- | --- |
| 总需求 | 12,000 件 |
| 第一批发货 | 9,000 件 |
| 第二批发货 | 3,000 件 |
| 最终交付状态 | 按最终交期完成 |
| 风险来源 | 供应商延期、焊接设备故障、备选供应商质量问题 |
| 成本影响 | 采购加急、加班、加严检验 |
| 质量影响 | 备选供应商批次隔离和追溯 |

经营分析 Agent 输出：

```text
本次订单通过启用备选供应商、加班和分批发运，最终维持客户最终交期。
主要代价是采购成本、加班成本和检验成本上升。
质量风险集中在 SUP-BETA-AL 首批铝板，已通过加严检验和隔离控制。
建议后续复盘主供应商交付稳定性，并评估 WLD-02 预测维护策略。
```

讲解要点：

- 管理层看到的是经营结果，不是单据列表。
- AESE 用这条故事证明 IAOS 可以支撑“运行、异常、决策、复盘”闭环。

## 7. Agent 验收

### 7.1 计划 Agent

必须回答：

- 总需求是多少。
- 净生产需求是多少。
- 铝板数量是否短缺。
- 铝板可用时点是否有风险。
- 哪个设备或工作中心造成产能风险。
- 推荐哪种方案。

最低可接受输出：

```text
总需求 12,000 件，扣除 1,200 件成品库存后需生产 10,800 件。
铝板数量上基本覆盖，但主供应商延期导致时点风险。
LAS-WLD-02 停机导致焊接产能风险。
建议释放备选供应商 PO、启用加班，并采用分批发运。
```

### 7.2 质量 Agent

必须回答：

- 哪个供应商批次出现质量问题。
- 缺陷是什么。
- 影响数量是多少。
- 是否允许直接投产。
- 建议的隔离和追溯动作是什么。

最低可接受输出：

```text
SUP-BETA-AL 的 BETA-20260712-01 批次在 IQC-202607-0002 中发现 300 张表面划伤。
建议隔离不合格数量，禁止直接投产；合格数量需质量放行后使用，并保留批次追溯。
```

### 7.3 经营分析 Agent

必须回答：

- 最终是否满足客户交期。
- 成本为什么上升。
- 风险如何被控制。
- 后续管理建议是什么。

最低可接受输出：

```text
通过备选供应商、加班和分批发运，最终维持客户最终交期。
成本上升来自加急采购、加班和加严检验。
质量风险通过批次隔离和追溯控制。
建议复盘主供应商交付稳定性和 WLD-02 设备维护策略。
```

## 9. 演示失败条件

出现以下情况则说明演示还不成立：

- 订单确认后没有事件链。
- 供应商延期或设备故障只是孤立记录，没有触发重算或风险解释。
- Agent 输出无法引用具体订单、采购订单、设备、批次或库存。
- 质量不良没有隔离或追溯建议。
- 管理层只能看到单据，无法看到经营影响。
- 没有 `correlation_id`，事件无法串成同一条故事。

## 10. 后续实现建议

建议后续开发顺序：

1. 先用 seed 事件重放方式实现这条故事，不要求所有用户操作都真实可点。
2. 再把订单确认、供应商延期、设备故障和检验失败做成可手工触发的操作。
3. 先做事件流视图和 Agent 只读建议。
4. 再做 Capability 草稿动作，例如释放备选供应商 PO、创建隔离质量问题、生成分批发运方案。
5. 最后做 2D 企业沙盘，把订单、产线、设备、库存和异常用可视化串起来。

这条演示故事完成后，AESE 才真正从“虚拟企业设定”进入“可运行企业仿真”阶段。
